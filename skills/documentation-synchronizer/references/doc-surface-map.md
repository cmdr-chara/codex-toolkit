# Documentation Surface Map

| Change class | Likely surfaces | Often missed |
| --- | --- | --- |
| API/schema/event | reference, schema descriptions, clients/examples, errors, migration/deprecation | generated sources, webhooks, event consumers, version negotiation |
| CLI | command reference, README, shell examples, completion, help snapshots, exit behavior | automation scripts and CI examples |
| Configuration | reference, sample config, env example, deployment charts/manifests, troubleshooting | default precedence, secret source, reload/restart behavior |
| Dependency/runtime | install, support matrix, development setup, CI/deploy images, migration guide | lockfile/toolchain pin and platform minimums |
| Data migration | operator guide, backup/restore, mixed-version behavior, rollback, release notes | irreversible point, reconciliation, cleanup criteria |
| UI/UX | user guide, labels, onboarding, empty/error states, screenshots, alt text | keyboard/screen-reader instructions and responsive variants |
| Architecture | overview, diagrams, ADRs, contributor guide, CODEOWNERS references | deployment/failure boundary and generated diagrams |
| Observability/ops | runbooks, dashboards, alerts, log/trace fields, incident steps | cardinality/privacy, post-release checks, rollback authority |
| Security/privacy | threat model, auth/permissions, data handling, admin/user docs | audit events, retention, consent, secrets examples |
