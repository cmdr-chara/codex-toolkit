# Expo and React Native Ecosystem Reference — 2026-07-17

**Information checked:** 2026-07-17  
**Next review:** 2026-10-15 or earlier after an Expo SDK, RN, store, or security release.  
**Baseline:** Expo SDK 57 (released 2026-06-30) maps to React Native 0.86, React 19.2.3, React Native Web 0.21.0, and minimum Node.js 22.13.x.  
**Platform table:** SDK 57 documentation lists Android 7+ with compile/target SDK 36 and iOS 16.4+ with Xcode 26.4+; recheck patch/tooling requirements at build time.  
**Architecture:** Expo SDK 55+ uses React Native's New Architecture permanently.

Use `npx expo install` for SDK-compatible Expo package versions when changing dependencies, then inspect the lockfile and native build. Do not force React or native package versions outside Expo's compatibility guidance.

## Known SDK 57 issue

Expo's SDK 57 changelog reports that importing Reanimated can raise memory use roughly 25–30% because of a Hermes V1 interaction. It documents Worklets bundle mode as a mitigation while a fix is developed. Measure the target app and recheck the changelog before release; do not generalize the percentage to every app or newer patch.

## Additional current guidance

- Expo's local-first guide is work in progress; evaluate sync/database products independently and do not mandate one.
- Duplicate React, React Native, Turbo, or Expo module versions in a monorepo can cause runtime/build errors; inspect dependency graph.
- EAS Submit requires signed store artifacts and does not replace store metadata/policy/account preparation.
